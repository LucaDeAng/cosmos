## 1. Tipi di Portfolio Item

### 1.1 Initiative (Iniziativa/Progetto)
**Definizione**: Un'attività temporanea con inizio e fine definiti, budget allocato, e deliverable specifici.

**Caratteristiche distintive**:
- Ha una data di inizio e fine
- Budget dedicato (CAPEX/OPEX)
- Team assegnato
- Milestone e deliverable definiti
- ROI misurabile post-completamento

**Esempi**:
- Migrazione al cloud AWS
- Implementazione SAP S/4HANA
- Sviluppo nuova app mobile
- Cyber security assessment
- Data warehouse modernization
- ERP upgrade
- Digital workplace transformation
- API platform implementation

**Keywords di riconoscimento** (IT/EN):
- progetto, project, iniziativa, initiative
- migrazione, migration, upgrade, implementazione
- trasformazione, transformation, modernizzazione
- sviluppo, development, deployment
- POC, proof of concept, pilota, pilot
- rollout, go-live, lancio

### 1.2 Product (Prodotto)
**Definizione**: Un asset software o piattaforma che l'azienda sviluppa/mantiene internamente o acquista, con ciclo di vita continuo.

**Caratteristiche distintive**:
- Ciclo di vita continuo (no end date)
- Ownership chiaro (Product Owner)
- Roadmap evolutiva
- Versionamento
- Utenti/clienti definiti
- Costi ricorrenti (manutenzione, licenze)

**Esempi**:
- Portale clienti B2B
- App mobile aziendale
- Piattaforma e-commerce
- Sistema di fatturazione interno
- Dashboard analytics
- Intranet aziendale
- Customer self-service portal
- Internal developer platform

**Keywords di riconoscimento** (IT/EN):
- prodotto, product, piattaforma, platform
- applicazione, application, app, software
- sistema, system, soluzione, solution
- portale, portal, dashboard
- tool, strumento, utility

### 1.3 Service (Servizio)
**Definizione**: Una capability IT erogata in modo continuativo, spesso da terze parti o team dedicati.

**Caratteristiche distintive**:
- Erogazione continua (24/7, SLA)
- Può essere interno o outsourced
- Misurabile tramite SLA/KPI
- Costo ricorrente (OPEX)
- Scalabile on-demand

**Esempi**:
- Managed firewall service
- Cloud hosting (IaaS)
- Service desk / helpdesk
- Backup as a service
- SAP managed service
- Network operations
- Security monitoring (SOC)
- Identity management service

**Keywords di riconoscimento** (IT/EN):
- servizio, service, managed service
- supporto, support, helpdesk, service desk
- hosting, SaaS, PaaS, IaaS, *aaS
- manutenzione, maintenance, operation
- monitoring, gestione, management
- outsourcing, MSP, provider

## 2. Categorizzazione per Valore Strategico

### 2.1 Run the Business (RTB)
**Descrizione**: Attività necessarie per mantenere le operazioni correnti.
- Obiettivo: Stabilità, affidabilità, compliance
- Budget tipico: 60-70% del budget IT
- Rischio se non fatto: Interruzione operativa

**Esempi**:
- Manutenzione ERP
- Rinnovo licenze
- Patch management
- Backup e DR
- Help desk

### 2.2 Grow the Business (GTB)
**Descrizione**: Attività che espandono le capability esistenti.
- Obiettivo: Incrementare efficienza, scalabilità
- Budget tipico: 20-30% del budget IT
- Rischio se non fatto: Perdita competitività graduale

**Esempi**:
- Upgrade infrastruttura
- Migrazione cloud
- Automazione processi
- BI e analytics
- Integration platform

### 2.3 Transform the Business (TTB)
**Descrizione**: Attività che cambiano il modello operativo o di business.
- Obiettivo: Innovazione, nuovi revenue stream
- Budget tipico: 10-20% del budget IT
- Rischio se non fatto: Disruption da competitor

**Esempi**:
- Digital transformation
- AI/ML initiatives
- New digital products
- Platform business models
- IoT/Edge computing

## 4. Lifecycle Stages

### 4.1 Initiative Lifecycle
```
Ideation → Assessment → Planning → Execution → Closure → Benefits Realization
```

| Stage | Status Values |
|-------|---------------|
| Ideation | proposed, idea, backlog |
| Assessment | under_review, feasibility |
| Planning | approved, planning |
| Execution | in_progress, active |
| Closure | completed, cancelled |
| Benefits | realized, measuring |

### 4.2 Product Lifecycle
```
Concept → Development → Launch → Growth → Maturity → Decline → Retirement
```

| Stage | Status Values |
|-------|---------------|
| Concept | proposed, poc |
| Development | in_development |
| Launch | beta, pilot |
| Growth | active, scaling |
| Maturity | stable, maintenance |
| Decline | sunset, deprecated |
| Retirement | retired, decommissioned |

### 4.3 Service Lifecycle
```
Strategy → Design → Transition → Operation → Improvement
```

| Stage | Status Values |
|-------|---------------|
| Strategy | proposed, planning |
| Design | design, build |
| Transition | pilot, rollout |
| Operation | active, stable |
| Improvement | optimizing |

## 7. Type Detection Heuristics

### Decision Tree per Type Detection

```
Il documento menziona...
│
├─ Date di inizio E fine specifiche?
│   ├─ SÌ → Probabilmente INITIATIVE
│   │   └─ Ha milestone/deliverable? → Conferma INITIATIVE
│   └─ NO → Continua...
│
├─ Parla di versioni, roadmap, product owner?
│   ├─ SÌ → Probabilmente PRODUCT
│   │   └─ Ha utenti/clienti definiti? → Conferma PRODUCT
│   └─ NO → Continua...
│
├─ Menziona SLA, erogazione continua, supporto?
│   ├─ SÌ → Probabilmente SERVICE
│   │   └─ Ha costi ricorrenti (mensili/annuali)? → Conferma SERVICE
│   └─ NO → Fallback...
│
└─ Default: Analizza keywords e contesto
    ├─ Più keywords "progetto/project" → INITIATIVE
    ├─ Più keywords "sistema/platform" → PRODUCT
    └─ Più keywords "servizio/support" → SERVICE
```

### Confidence Scoring
| Confidence | Criteria |
|------------|----------|
| High (>80%) | 3+ indicatori chiari per un tipo |
| Medium (50-80%) | 2 indicatori o mix di segnali |
| Low (<50%) | Solo keywords, nessun indicatore strutturale |

## 8. Budget Classification

### 8.1 CAPEX vs OPEX
| CAPEX (Capital Expenditure) | OPEX (Operational Expenditure) |
|-----------------------------|--------------------------------|
| Software licenses (perpetual) | SaaS subscriptions |
| Hardware purchase | Cloud IaaS/PaaS |
| Custom development | Managed services |
| Implementation projects | Maintenance contracts |
| Infrastructure build | Support agreements |

### 8.2 Budget Ranges (Indicative)
| Size | Budget Range | Typical Duration |
|------|--------------|------------------|
| Small | <€50K | <3 mesi |
| Medium | €50K-€250K | 3-6 mesi |
| Large | €250K-€1M | 6-12 mesi |
| Strategic | >€1M | >12 mesi |
